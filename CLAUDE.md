# saas-kit — Claude Code Context

This repo contains `setup.sh`, a single-script installer for a self-hosted SaaS stack on Ubuntu 24.04 LTS, built on top of vps-secure.

## What setup.sh installs

| Service | Container | Host Port | Internal Port |
|---|---|---|---|
| PostgreSQL 16 | `saaskit-postgres` | none (internal) | 5432 |
| Dragonfly | `saaskit-dragonfly` | none (internal) | 6379 |
| Redis 7 | `saaskit-redis` | none (internal) | 6379 |
| n8n | `saaskit-n8n` | 127.0.0.1:5678 | 5678 |
| n8n-MCP | `saaskit-n8n-mcp` | 127.0.0.1:5679 | 3000 |
| Baserow | `saaskit-baserow` | 127.0.0.1:5680 | 80 |
| MinIO API | `saaskit-minio` | 127.0.0.1:9000 | 9000 |
| MinIO Console | `saaskit-minio` | 127.0.0.1:9001 | 9001 |

Caddy is NOT managed by this stack — it runs as `vps-monitor-caddy` (network_mode:host), installed by vps-secure's `install-dashboard.sh`. setup.sh injects reverse proxy blocks into `/home/vpsadmin/vps-monitor/Caddyfile`.

## Filesystem layout

```
/opt/saas-kit/
├── .env                          # generated — chmod 600 — DO NOT COMMIT
├── docker-compose.yml            # generated by setup.sh
├── Caddyfile                     # NOT here — see /home/vpsadmin/vps-monitor/Caddyfile
├── initdb/
│   └── 01-create-databases.sql  # creates n8n_db and baserow_db
├── templates/
│   ├── awesome-n8n-templates/   # n8n workflow JSON templates
│   └── n8n-skills/              # Claude Code skillset for n8n workflows
└── data/                        # all persistent volumes
    ├── postgres/
    ├── dragonfly/
    ├── redis/
    ├── n8n/
    ├── baserow/
    └── minio/
```

## Credentials

All credentials are stored in `/etc/vps-secure/saas-kit.conf` (chmod 600).  
Never hardcode credentials — always read from that file.

```bash
# Read a specific value
grep N8N_PASSWORD /etc/vps-secure/saas-kit.conf | cut -d'"' -f2
```

## Docker network

All saas-kit containers share `saaskit-net` (bridge). They communicate by container name:
- n8n → postgres: `postgres:5432`
- n8n → dragonfly: `dragonfly:6379`
- n8n-mcp → n8n: `saaskit-n8n:5678`
- baserow → postgres: `postgres:5432`
- baserow → redis: `redis:6379`

## PostgreSQL databases

| Database | Owner | Used by |
|---|---|---|
| `n8n_db` | `saaskit` | n8n |
| `baserow_db` | `saaskit` | Baserow |

Connect from host:
```bash
docker exec -it saaskit-postgres psql -U saaskit -d n8n_db
```

## Key commands

```bash
cd /opt/saas-kit

# Status
docker compose ps

# Logs
docker compose logs -f [service]   # service = postgres|dragonfly|redis|n8n|n8n-mcp|baserow|minio

# Restart a service
docker compose restart [service]

# Full restart
docker compose down && docker compose --env-file .env up -d

# Configure n8n API key for n8n-MCP (run after getting key from n8n UI)
sudo saaskit-mcp-apikey.sh <N8N_API_KEY>
```

## Post-install checklist

1. **n8n-MCP** — go to `https://n8n.<domain>` → Settings → API → Create API Key → `sudo saaskit-mcp-apikey.sh <key>`
2. **Baserow** — go to `https://baserow.<domain>` → register with admin email
3. **MinIO** — go to `https://minio-console.<domain>` → verify login
